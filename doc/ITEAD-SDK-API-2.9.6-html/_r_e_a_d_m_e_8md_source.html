7 # What Is It?
8 
9 ITEAD-SDK is an open-source interface library for hardware manipulation
10 based on Linux platform and compatible with [Arduino API][1].
11 You can develop with it by using C/C++ and Python. ITEAD-SDK is dedicated to provide
12 a fast, efficient and unified software development environment for the person
13 like you with a strong electronic design idea.
14 
15 # What Can It Do?
16 
17 Through ITEAD-SDK, users can directly access to the Linux platform hardware
18 resources such as GPIO, UART, SPI, I2C etc. ITEAD-SDK supports various
19 hardware platforms, such as Iteaduino Plus A10/A20, IBOX,
20 ITEAD Core EVB, Raspberry Pi and Raspberry Pi Model B+ and so on. Meanwhile, It also
21 provides a great amount of libraries for all kinds of child board like NFC(PN532)
22 module, GPS shield, GSM(SIM900/SIM908) shield, nRF24L01 module, OLED and LCD module etc.
23 
24 Simplifying the hardware operation makes it possible for software engineers
25 who are not familiar with the underlying hardware and junior developers to
26 understand more about "Code World" and "Physical World". It can also work
27 as a rapid prototyping development tool.
28 
29 # Features
30 
31 At present, ITEAD-SDK has a set of **core features** as below:
32 
33  - Support for external interrupts operations
34  - Support for upper-layer time functions
35  - Support for PWM output with any GPIO
36  - Support for GPIO operations
37  - Support for GPIO 8bit/16bit bus operations
38  - Support for UART bus operations
39  - Support for I2C bus operations
40  - Support for SPI bus operations
41 
42 Based on core features, plenty of **functional features** can be reached:
43 
44  - Support for SSD1306-based I2C OLED screen operations
45  - Support for 1602 LiquidCrystal module
46  - Support for GPS module
47  - Support for GSM(SIM900/SIM908) module including GPRS/GPS operations
48  - Support for NFC(PN532) module
49  - Support for nRF24L01 module
50  - Support for Temperature and Humidity module

185 # How to Start
186 
187 [ITEAD-SDK Python API][2] will be recommanded for Python users.
188 SDK works well with C(.c) and C++(.cpp) programs. Especially, it is
189 compatiable with Arduino Sketch(.ino).
190 
191 ## To use SDK in Python
192 
193 Create a new file named `blink.py` which reads as follow:
194 
195  #! /usr/bin/env python
196  from iteadsdk import *
197  LED = PG9
198  pinMode(LED, OUTPUT)
199  while True:
200  digitalWrite(LED, HIGH)
201  delay(1000)
202  digitalWrite(LED, LOW)
203  delay(1000)
204 
205 Make `blink.py` executable:
206 
207  $ chmod +x blink.py
208 
209 Then, run it:
210 
211  $ sudo ./blink.py
212 
213 You can get commutative HIGH and LOW from GPIO PG9, if you've run it on ITEAD CORE EVB
214 or other platforms based on Allwinner A10/A20 SoC.
215 
216 ## To use SDK in C program
217 
218 Create a new file named "led.c" which reads as follow:
219 
220  #include <itead.h>
221  #include <stdio.h>
222 
223  #define STATUS_LED1 PG9
224 
225  void setup(void)
226  {
227  printf("setup begin\n");
228  pinMode(STATUS_LED1,OUTPUT);
229  }
230  void loop(void)
231  {
232  digitalWrite(STATUS_LED1,HIGH);
233  delay(1000);
234  digitalWrite(STATUS_LED1,LOW);
235  delay(1000);
236  }
237  void main(int argc, char **argv)
238  {
239  setup();
240  while(1)
241  {
242  loop();
243  }
244  }
245 
246 Then run iteadcompile in the terminal
247 
248  $ iteadcompile LED led.c
249 
250 You can run the LED program after compilation:
251 
252  $ sudo ./LED

324 #### Note: How to understand debug information from compiler
325 
326 You DO NOT include the itead.h explicitly in your source file with suffix .ino.
327 iteadcompile will take care of everything. That means you need to know the method
328 you should use for locating your .ino sketch bugs. Actually,if you have a sketch
329 named "Hello.ino" with a bug at level ERROR in line 25, you will get some ERROR
330 informations telling the ERROR bug in "Hello.cpp" with line 26(25+1) when
331 command "iteadcompile Hello Hello.ino" is executed. Why? iteadcompile insert one
332 line like `include "itead.h"` in the head of your Hello.ino, change the suffix
333 .ino to .cpp and compile(using g++) Hello.cpp to generate Hello which is
334 executable binary file. When you encounter other bugs in your sketch,debug
335 skills on C++ program will also helpful for solving your problems.
